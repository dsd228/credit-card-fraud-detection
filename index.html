🔍 Detección de Fraude con Tarjetas de Crédito

Análisis y modelado de datos reales para identificar actividades fraudulentas usando Machine Learning.

Ver Código en GitHub

Datos

El dataset proviene de Kaggle y contiene 284,807 transacciones, de las cuales solo 492 son fraudes (0.17%).

  • Datos anónimos y normalizados
  • Variable objetivo: Class (1 = Fraude, 0 = No Fraude)
  • Desbalance significativo en las clases

Análisis Exploratorio

Se analizaron las correlaciones entre variables, distribuciones, valores atípicos y se visualizó la desproporción de clases.

Modelos

Se entrenaron distintos algoritmos para evaluar su rendimiento frente a un conjunto de prueba:

  • Regresión Logística
  • Random Forest
  • XGBoost

Se evaluaron con las métricas: precisión, recall, F1-score y AUC.

Resultados

Se muestran a continuación los resultados obtenidos por el modelo Random Forest:

  • Accuracy: 99.92%
  • Recall (Fraudes detectados): 89%
  • Precision: 93%
  • AUC: 0.98

Conclusión

El modelo basado en Random Forest logró resultados sólidos ante un dataset altamente desbalanceado. Se podrían aplicar técnicas de oversampling (SMOTE) o modelos secuenciales (LSTM) para mejorar aún más el recall sin perder precisión.
