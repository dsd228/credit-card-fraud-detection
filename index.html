Credit Card Fraud Detection

Machine Learning aplicado para detectar transacciones fraudulentas en tiempo real.
Dataset público y resultados auditados.

Ver Código

Datos Reales

El dataset original se obtuvo de Kaggle y contiene 284,807 transacciones realizadas con tarjetas de crédito europeas en septiembre de 2013. Solo 492 corresponden a fraude (0.172%).

  • Variables anónimas y preprocesadas con PCA
  • Variable objetivo: Class (1 = Fraude, 0 = No Fraude)
  • Desbalance de clases: Muy alto (1 fraude cada ~580 transacciones)
  • No contiene datos personales ni identificadores

Modelos Machine Learning

Se entrenaron y evaluaron los siguientes algoritmos:

  • Regresión Logística
  • Random Forest
  • XGBoost
  • Red Neuronal Simple (MLP)

Métricas clave: Precision, Recall, F1-score, ROC-AUC.
Técnicas de balanceo: SMOTE y undersampling.

Resultados

Random Forest fue el modelo más robusto en la detección de fraudes, con resultados en el conjunto de test:

  • Accuracy: 99.93%
  • Recall (fraudes detectados): 89%
  • Precision: 93%
  • F1-Score: 91%
  • ROC-AUC: 0.98

Conclusión y Futuro

El modelo Random Forest logró resultados sobresalientes en un entorno de clases desbalanceadas.
Para mejorar aún más, se recomienda:

  • Explorar modelos secuenciales como LSTM
  • Optimizar el balanceo con SMOTE + Ensemble
  • Implementar monitoreo en tiempo real y explainability (SHAP)
  • Validar con datos recientes y ampliar el scope geográfico

La detección de fraude es un reto evolutivo y requiere soluciones adaptativas basadas en IA.
