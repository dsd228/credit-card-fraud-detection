Credit Card Fraud Detection

Un proyecto de Machine Learning para la detección de transacciones fraudulentas en tiempo real, utilizando un dataset público y técnicas avanzadas de modelado.

Ver Código en GitHub

El Dataset

El proyecto utiliza el dataset "Credit Card Fraud Detection" de Kaggle. Contiene 284,807 transacciones, de las cuales solo 492 (0.172%) son fraudulentas. Este severo desbalance de clases es el principal desafío del problema.

Análisis Exploratorio (EDA)

El análisis inicial reveló que las variables (V1-V28, resultado de PCA) no tienen correlaciones obvias, y que los montos de transacciones fraudulentas tienden a ser más bajos. Visualizar los datos en 2D con PCA ayuda a ver una ligera separación entre clases.

Modelado y Estrategia

Se probaron varios algoritmos, pero Random Forest demostró ser el más efectivo. Para manejar el desbalance de clases, se aplicó la técnica SMOTE (Synthetic Minority Over-sampling Technique) en el set de entrenamiento.

Resultados del Modelo

El modelo Random Forest con SMOTE alcanzó un rendimiento excelente en el conjunto de prueba:

  • Accuracy: 99.95%
  • Recall (Sensibilidad): 89%
  • Precision: 92%
  • F1-Score: 0.90
  • ROC-AUC: 0.97

Conclusión y Próximos Pasos

El modelo es altamente eficaz, pero siempre hay espacio para mejorar. Los próximos pasos podrían incluir:

  • Explainability (XAI): Usar SHAP para entender las predicciones.
  • Modelos más complejos: Probar redes neuronales (LSTM) para capturar secuencias.
  • Deployment: Empaquetar el modelo en una API para predicciones en tiempo real.
